# Progressive Disclosure and Pacing

## Core Findings

### The Generating Principle

Progressive disclosure is not information rationing. It is the sequential construction of the interpretive context required for information to be meaningful. Each element in a sequence delivers content AND transforms the interpretive frame for everything after it. [VERIFIED -- cross_domain_pattern_synthesis.md, Generating Principles #3 and #12]

This reframes progressive disclosure from a UX pattern ("show less, reveal more on demand") to a structural principle: the ORDER of revelation determines the MEANING of what is revealed.

### Pacing as Material (Operator-Confirmed)

Progressive disclosure with dimming (healthy curve fades to gray when disrupted curve appears) is the right teaching pattern. The 1.8s pause before "It's when" works as a beat drop. Staggered bar reveals work. "Unless the rhythm never comes down" as transition text works. [TESTED -- operator confirmed these specific patterns unprompted as what they liked]

Future animations should use these patterns: dim prior content when introducing contrasting content, use deliberate pauses before reveals, stage information progressively. Pacing is material, not just motion. [TESTED -- feedback-animation-pacing.md]

### Timing, Transitions, and Floor

Scene-to-scene transitions, within-scene element timing, and overall structure are confirmed at a high quality floor. The floor itself is high -- the remaining delta is taste/judgment on content and framing, not execution quality of motion. [TESTED -- feedback-animation-transitions.md, operator confirmation]

Baseline standard: Remotion frame-perfect rendering, deliberate pauses between reveals, staggered element entrances, progressive disclosure with dimming. Do not regress below this level. [TESTED]

### The Loading Principle

Value is created during preparation, not during display. The loading state is a product, not overhead. [VERIFIED -- cross_domain_pattern_synthesis.md, Meta-Pattern #2]

Applied: a calculator loading state should show calculation dimensions resolving (what the system is doing), not a spinner (that the system is busy). The loading experience IS the value -- it builds anticipation and frames the result. [VERIFIED -- cross_domain_pattern_synthesis.md]

The threshold between states is its own state, lasting 300-600ms, where the system visibly works. This is not delay -- it is the moment where the user's interpretive frame shifts from "I asked a question" to "the answer is coming." [VERIFIED -- cross_domain_pattern_synthesis.md]

### Hitchcock's Bomb Applied to Progressive Disclosure

Give the audience information the characters don't have. Suspense is generated by strategic withholding. A landing page that shows everything immediately gives 15 seconds of surprise. A landing page that reveals its most important element only after a scroll, interaction, or beat of anticipation builds suspense. [VERIFIED -- Hitchcock AFI/Truffaut interviews, confirmed in directing_editorial_pacing.md]

**But the bomb must not go off:** If you build anticipation and fail to deliver meaningful resolution, the audience feels cheated. Suspense must resolve with payoff, not abandonment. A calculator that builds up to a result with a loading animation and then returns "moderate risk" without explanation is the bomb going off. [VERIFIED -- directing_editorial_pacing.md]

### The Kuleshov Effect in Sequences

An image does not carry inherent meaning. Its meaning is constructed by the images adjacent to it in sequence. Every screen is "cut" with the screen the user just left -- the prior screen's emotional register infects the current one. [VERIFIED -- Kuleshov experiment, PMC NIH behavioral research, directing_editorial_pacing.md]

**The Kuleshov Trap:** Every animation adds a contextual frame that colors the meaning of what follows. A celebratory particle effect on a BMI result classifying someone as obese does not just fail -- it creates the wrong emotional sequence. The Kuleshov effect means the user arrives at the number inside that celebratory frame. There are no neutral animations. [THEORETICAL -- inferred from Kuleshov applied to UI]

### The Eight Editorial Decisions in a Sequence

Every progressive disclosure sequence involves eight editorial decisions: [VERIFIED for film principles; THEORETICAL for UI application]

1. **Opening frame** -- what does the user see before interaction? Establishes emotional register and implied promise.
2. **What is withheld** -- strategic absence creates anticipation (Hitchcock's bomb).
3. **What triggers the next reveal** -- scroll (earned), timer (wait for it), click (chosen), hover (curiosity rewarded). Each trigger is an editorial statement.
4. **Rhythm of reveals** -- equal intervals = mechanical; accelerating = urgency; decelerating at the important moment = gravity.
5. **Emotional register of each state** -- form state = uncertain/anticipatory; loading state = trust/patience; result state = revelation/clarity.
6. **Cut type between states** -- hard cut (decisive), dissolve (one becomes the other), match cut/FLIP (these are the same object).
7. **Payoff frame** -- the editorial climax; Hitchcock's Rule says the most important element gets the most space.
8. **Exit state** -- where does the sequence leave the user? Resolved? Open question? Something to share?

### Staging Hierarchy

One thing should capture attention at a time. Multiple elements animating simultaneously create visual noise. [VERIFIED -- NN/G, UX Collective, animation_12_principles.md]

Hierarchy rules:
1. Movement beats stillness -- a moving element wins attention over static ones
2. Contrast beats sameness -- bright on dark, large on small, fast on slow
3. Sequence beats simultaneous -- enter elements in reading order
4. Primary action gets full motion; supporting elements get fade-only
5. Never animate more than 2-3 elements in the same 300ms window

[VERIFIED -- animation_12_principles.md]

### Dimming as Disclosure Mechanism

When introducing contrasting content, dim prior content (fade to gray, reduce opacity) rather than removing it. This preserves context while clearly signaling "attention should shift here." The prior content becomes background; the new content becomes foreground. [TESTED -- operator confirmed this specific pattern]

This is the visual equivalent of Eisenstein's tonal montage: the emotional register of each frame is deliberate, and the contrast between frames carries meaning. [THEORETICAL -- synthesized from directing_editorial_pacing.md]

### The Beat Drop

A deliberate pause (1.5-2.0s) inserted before the most important reveal in a sequence. The pause says "this is important, wait for it." In traditional animation, a held pose is as expressive as the motion itself. [VERIFIED -- animation_12_principles.md; TESTED -- 1.8s pause confirmed as effective by operator]

In CSS keyframes, the beat drop is a hold: 40% to 65% at the same values, then the reveal at 65%. The pause from 40%-65% is doing work -- it is saying "read this, process it, now here comes the next thing." [VERIFIED -- animation_12_principles.md]

### Pink Noise Pacing

Rhythm is the management of prediction. Entrain, then violate, then resolve -- at multiple timescales simultaneously. The optimal distribution is 1/f (pink noise): variation at all timescales, never perfectly regular, never fully random. [VERIFIED -- cross_domain_pattern_synthesis.md, Generating Principle #5]

If reveals happen at exactly equal intervals, the pacing feels mechanical (metric montage). If they accelerate, they create urgency. If they decelerate at the most important moment, they create gravity. Murch's Rule: rhythm is worth 10% of the edit -- not nothing, but subordinate to emotion (51%) and story (23%). [VERIFIED -- directing_editorial_pacing.md]

### Three-Suspension Limit

Never have more than 3 unresolved conceptual threads active simultaneously. Beyond 3, the viewer loses track and the sequence collapses from tension into confusion. [VERIFIED -- cross_domain_pattern_synthesis.md]

### Nucleation

The first element in any sequence contains the structural geometry of everything that follows. If the first element sets the wrong frame, every subsequent reveal is colored by that error. Design the first reveal to establish the interpretive frame explicitly. [VERIFIED -- cross_domain_pattern_synthesis.md]

### The Match Cut as Causality

When a button expands into a results card (FLIP animation), the button and card share spatial identity -- they are the "same object" across two states. This creates the cognitive message: "This result came from your input." Without the match cut relationship, results feel disconnected from the action. [VERIFIED -- directing_editorial_pacing.md]

### Tempo and Rate

Systems respond to the derivative, not the state. First derivative (rate of change) captures attention. Second derivative (acceleration) encodes emotion. Third derivative (jerk) triggers arousal. [VERIFIED -- cross_domain_pattern_synthesis.md]

**Design the acceleration curve, not the duration.** Deceleration carries more information than acceleration. The hold (fermata) is the loudest moment. Match rate to emotional arc, not informational arc. [VERIFIED -- cross_domain_pattern_synthesis.md]

## Operational Rules

1. **When revealing contrasting content, dim prior content to gray/reduced opacity rather than removing it** -- because dimming preserves context while clearly signaling the attention shift, and this pattern is operator-confirmed as the right teaching approach.

2. **When a sequence has a primary payoff (result number, key insight), insert a 1.5-2.0s beat drop pause before it** -- because the deliberate pause creates anticipation and signals importance, confirmed at 1.8s as effective.

3. **When building a loading state, show what the system is computing, not that it is busy** -- because the loading experience IS value, building the interpretive frame for the result (the Loading Principle).

4. **When sequencing progressive reveals, ensure each reveal transforms the interpretive frame for the next** -- because progressive disclosure is frame construction, not information rationing; order determines meaning.

5. **When choosing animation for a result reveal, verify the emotional register matches the content** -- because the Kuleshov effect means a celebratory animation on bad health data creates mockery, not delight; there are no neutral animations.

6. **When pacing a multi-step sequence, never exceed 3 unresolved conceptual threads simultaneously** -- because beyond 3, viewers lose track and the sequence collapses from tension into confusion.

7. **When a result appears after user input, use FLIP/match-cut to morph the input element into the result** -- because the shared spatial identity creates perceived causality ("this came from what I entered").

8. **Do not regress below the confirmed animation floor: frame-perfect rendering, deliberate pauses, staggered entrances, progressive disclosure with dimming** -- because this floor is operator-confirmed as high, and the remaining delta is taste, not execution.
